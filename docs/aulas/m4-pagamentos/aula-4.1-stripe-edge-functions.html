Checkout, webhook e planos.
O CRM começa a faturar.

O Stripe cuida do pagamento. Uma Edge Function do Supabase recebe a confirmação e ativa o plano automaticamente. Separação limpa: Next.js cuida da interface, Supabase cuida do dinheiro.

O que essa aula constrói

Quatro peças. Receita ativada.

Checkout via Stripe, webhook via Edge Function, limites por plano e portal do cliente. Tudo que um SaaS precisa pra cobrar.

💳

Stripe Checkout

Botão 'Assinar Pro' redireciona pro Stripe. Eles hospedam o pagamento — cartão, Pix, tudo pronto.

⚡

Edge Function (Webhook)

Quando o Stripe confirma o pagamento, manda um POST pra Edge Function do Supabase. Ela ativa o plano no banco.

🔒

Limites de Plano

Free: 50 leads e 2 membros. Pro: ilimitado. A função canAddLead() e canAddMember() controlam tudo.

🎛️

Customer Portal

Página pronta do Stripe pra gerenciar assinatura. Trocar cartão, cancelar, nota fiscal — sem código extra.

Os prompts da aula

Do planejamento ao deploy

Prompt 01 — Planejamento

Criar a branch e alinhar com o Claude o que vai ser feito.

Leia o CLAUDE.md e o PLAN.md. Estamos na aula 4.1 — Monetização com Stripe via Supabase Edge Functions. Crie a branch feat/billing e me descreva o que vai implementar.
Prompt 02 — Chaves do Stripe

Configurar as variáveis de ambiente do Stripe.

Adicione no .env.local as chaves do Stripe: STRIPE_SECRET_KEY=[cola aqui] NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=[cola aqui] STRIPE_PRO_PRICE_ID=[cola aqui]
Prompt 03 — Checkout (Server Action)

Implementar o checkout do Stripe usando Server Action do Next.js.

Implemente o checkout do Stripe usando Server Action: - Server Action createCheckoutSession() em src/app/actions/stripe.ts - Cria uma Checkout Session com o STRIPE_PRO_PRICE_ID - Passa workspace_id e user_id como metadata - success_url e cancel_url apontando pra /settings/billing - Retorna a URL do Stripe pro frontend redirecionar
Prompt 04 — Edge Function (Webhook)

Criar a Edge Function do Supabase para receber eventos do Stripe.

Crie a Edge Function do Supabase para o webhook do Stripe: 1. npx supabase init (se ainda não inicializado) 2. npx supabase functions new stripe-webhook 3. Implemente supabase/functions/stripe-webhook/index.ts: - Verificar assinatura do Stripe (STRIPE_WEBHOOK_SECRET) - checkout.session.completed → ativar plano Pro (upsert subscriptions) - customer.subscription.deleted → voltar pro Free - invoice.payment_failed → marcar como payment_failed - Usar service_role key pra acessar o banco - É Deno, não Node.js. Usar Deno.serve() e imports de https://esm.sh/
Prompt 05 — Secrets e Teste Local

Configurar secrets e testar o webhook localmente com Stripe CLI.

Configure os secrets da Edge Function: npx supabase secrets set STRIPE_SECRET_KEY=sk_test_... STRIPE_WEBHOOK_SECRET=whsec_... Para testar local: npx supabase functions serve stripe-webhook --env-file supabase/.env.local stripe listen --forward-to http://127.0.0.1:54321/functions/v1/stripe-webhook
Prompt 06 — Limites e Billing Page

Implementar limites de plano e a página de cobrança.

Implemente os limites de plano e a página de cobrança: - Plano Free: máximo 50 leads e 2 membros. Mostrar aviso ao atingir limite. - Página /settings/billing com plano atual, comparação Free vs Pro, botão assinar - Botão "Gerenciar Assinatura" abre o Customer Portal do Stripe - canAddLead() e canAddMember() em src/lib/limits.ts
Prompt 07 — Deploy e Commit

Deploy da Edge Function, commit final e merge.

Faça o deploy da Edge Function (npx supabase functions deploy stripe-webhook), commit, PR pra main, merge e delete a branch.
