Os prompts da aula

Do planejamento ao webhook

Prompt 01 — Planejamento

Criar a branch e alinhar com o Claude o que vai ser feito.

Leia o CLAUDE.md e o PLAN.md. Estamos na aula 4.1 — Checkout & Webhook com Stripe. Crie a branch feat/billing e me descreva o que vai implementar.
Prompt 02 — Chaves .env

Adicionar as chaves do Stripe no projeto. Copie do Dashboard.

Adicione no .env.local as chaves do Stripe: STRIPE_SECRET_KEY=[cola aqui] NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=[cola aqui] STRIPE_PRO_PRICE_ID=[cola aqui]
Prompt 03 — Checkout + Webhook

O prompt principal — cria o checkout pra cobrar e o webhook pra ativar.

Implemente a monetização com Stripe conforme o PLAN.md: - Botão "Assinar Pro" que redireciona pro checkout do Stripe - Webhook que recebe a confirmação do Stripe e ativa o plano Pro no banco - Quando cancelar, webhook volta o plano pra Free - O webhook é a única exceção que usa Route Handler — todo o resto usa Server Actions
i

O webhook é a ÚNICA exceção que usa Route Handler no projeto inteiro. Todos os outros endpoints usam Server Actions. Webhooks precisam de Route Handler porque são chamados externamente pelo Stripe — não passam pelo browser do usuário.

Prompt 04 — Webhook Secret

Adicionar o secret do webhook depois de configurar no Stripe.

Adicione no .env.local a chave do webhook: STRIPE_WEBHOOK_SECRET=[cola aqui]
i

Cartão de teste: 4242 4242 4242 4242. Qualquer data futura e qualquer CVC de 3 dígitos. Zero cobrança real em modo teste.

Prompt 05 — Revisão

Claude revisa o próprio trabalho antes do teste humano.

Revise o que você criou. Confirme que o checkout redireciona pro Stripe, que o webhook ativa o plano no banco, que as chaves estão protegidas no .env.local e que o build passa sem erros.
Prompt 06 — Commit intermediário

Salvar o progresso antes de continuar com billing na próxima aula.

Faça um commit intermediário com o checkout e webhook do Stripe.
