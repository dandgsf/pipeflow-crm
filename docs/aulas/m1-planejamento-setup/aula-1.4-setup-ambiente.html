Skills, Agents & Plugins

AI Templates — Superpoderes do Claude Code

O que é: O Claude Code sozinho já é poderoso, mas com skills, agents e plugins ele se torna imbatível. Skills ensinam boas práticas, agents são especialistas em domínios específicos, e plugins conectam a serviços externos. Vamos instalar tudo pelo AI Templates (aitmpl.com) — o marketplace de componentes para Claude Code.
AI Templates (aitmpl.com)

Onde o Claude Code guarda as skills e agents?

O Claude Code usa dois diretórios diferentes dependendo do escopo:

Global — disponível em TODOS os projetos

Diretório:

Caminho ~/.claude/skills/<nome-da-skill>/SKILL.md
Caminho ~/.claude/agents/<nome-do-agent>/agent.md

Windows %USERPROFILE%\.claude\   Mac/Linux ~/.claude/

Tudo que estiver aqui fica acessível em qualquer projeto que você abrir no Claude Code.

Local — disponível SÓ neste projeto

Diretório (na raiz do projeto):

Caminho .claude/skills/<nome-da-skill>/SKILL.md
Caminho .claude/agents/<nome-do-agent>/agent.md

Tudo que estiver aqui fica acessível apenas dentro deste repositório. Pode ser commitado no Git e compartilhado com o time.

No terminal (fora do Claude Code), rode o AI Templates para instalar via marketplace:

Terminal npx claude-code-templates@latest

Isso abre o marketplace onde você pode explorar e instalar componentes. Abaixo vamos instalar cada um direto pelo terminal.

Instale as Skills do PipeFlow — cada uma ensina o Claude a ser melhor em uma área:

Skill 1 — Frontend Design

Cria interfaces distintas e production-grade. Foge da estética genérica de IA.

Global (~/.claude/skills/) npx claude-code-templates@latest --agent development-team/frontend-developer --yes
Local (.claude/skills/ do projeto) cd ~/pipeflow-crm && npx claude-code-templates@latest --agent development-team/frontend-developer --yes --local

Skill 2 — Supabase Postgres Best Practices

40+ regras de otimização: query performance, RLS, connection pooling, schema design.

Global (~/.claude/skills/) npx superpowers@latest install supabase-postgres-best-practices
Local (.claude/skills/ do projeto) cd ~/pipeflow-crm && npx superpowers@latest install supabase-postgres-best-practices --local

Skill 3 — Stripe Best Practices

Padrões para integração Stripe: checkout flows, webhooks seguros, subscriptions e idempotency.

Global (~/.claude/skills/) npx superpowers@latest install stripe-best-practices
Local (.claude/skills/ do projeto) cd ~/pipeflow-crm && npx superpowers@latest install stripe-best-practices --local

Skill 4 — Security Audit

Auditoria de segurança com análise de RLS, vulnerabilidades, validação de inputs e OWASP.

Global (~/.claude/skills/) npx superpowers@latest install supabase-security-audit
Local (.claude/skills/ do projeto) cd ~/pipeflow-crm && npx superpowers@latest install supabase-security-audit --local

Skill 5 — Micro SaaS Launcher

Guia para lançar SaaS rápido: MVP scoping, pricing, launch playbooks e tração inicial.

Global (~/.claude/skills/) npx superpowers@latest install micro-saas-launcher
Local (.claude/skills/ do projeto) cd ~/pipeflow-crm && npx superpowers@latest install micro-saas-launcher --local

Instale os Agents do PipeFlow — especialistas que trabalham junto com o Claude:

Agent 1 — Code Review

Revisa código contra o plano original, padrões de qualidade, bugs e vulnerabilidades.

Global (~/.claude/agents/) npx claude-code-templates@latest --agent development-tools/code-reviewer --yes
Local (.claude/agents/ do projeto) cd ~/pipeflow-crm && npx claude-code-templates@latest --agent development-tools/code-reviewer --yes --local

Agent 2 — Explore

Explorador rápido de codebases: busca arquivos, padrões, endpoints e arquitetura.

Global (~/.claude/agents/) npx claude-code-templates@latest --agent development-team/frontend-developer --yes
Local (.claude/agents/ do projeto) cd ~/pipeflow-crm && npx claude-code-templates@latest --agent development-team/frontend-developer --yes --local

Agent 3 — Plan

Arquiteto que desenha planos de implementação e identifica arquivos críticos.

Global (~/.claude/agents/) npx claude-code-templates@latest --agent development-tools/code-reviewer --yes
Local (.claude/agents/ do projeto) cd ~/pipeflow-crm && npx claude-code-templates@latest --agent development-tools/code-reviewer --yes --local

Instale os Plugins (MCPs) do PipeFlow — integrações com serviços externos:

Plugin 1 — Vercel

Deploy, env vars, domains, logs e best practices de Next.js automáticos.

Global (~/.claude/settings.json) npx claude-code-templates@latest --mcp development/github-integration --yes
Local (.claude/settings.json do projeto) cd ~/pipeflow-crm && npx claude-code-templates@latest --mcp development/github-integration --yes --local

Plugin 2 — Stripe

Acesso direto à API Stripe: explica erros, test cards e best practices de pagamento.

Global (~/.claude/settings.json) npx claude-code-templates@latest --mcp database/postgresql-integration --yes
Local (.claude/settings.json do projeto) cd ~/pipeflow-crm && npx claude-code-templates@latest --mcp database/postgresql-integration --yes --local

Verificação — rode o health check para confirmar tudo:

Terminal npx claude-code-templates@latest --health-check
✓

Se aparecer tudo verde, skills, agents e plugins estão instalados e prontos.

!

Resumo dos diretórios:
• Global = ~/.claude/skills/ e ~/.claude/agents/ — vale pra TODOS os projetos
• Local = .claude/skills/ e .claude/agents/ (na raiz do projeto) — vale SÓ pra esse projeto
• MCPs/Plugins = configurados em ~/.claude/settings.json (global) ou .claude/settings.json (local)

i

Dica: Você pode explorar mais de 600 componentes no marketplace em aitmpl.com. Ao longo do curso, a gente pode instalar mais conforme necessidade.

Supabase — Banco de Dados

O que é: Supabase é o nosso banco de dados + autenticação + segurança. Tudo num lugar só, com plano gratuito generoso. Usa PostgreSQL por baixo dos panos.
Criar conta no Supabase

Acesse o link e clique em "Sign Up with GitHub"

Mais prático, já que você criou a conta na Etapa 3.

Autorize o acesso do Supabase ao seu GitHub

Você cai no dashboard → clique em "New Project"

Preencha os dados do projeto:

• Name: pipeflow-crm

• Database Password: crie uma senha forte (anote em lugar seguro!)

• Region: escolha South America (São Paulo) para menor latência

Clique em "Create new project" e aguarde 1-2 minutos

✓

Projeto criado! Anote a URL e a anon key em Settings → API.

"Supabase criado. Banco de dados PostgreSQL rodando. A gente vai configurar as tabelas no módulo 3."

Stripe — Pagamentos

O que é: Stripe é o processador de pagamentos. Checkout, assinaturas, webhooks — tudo passa por ele. Vamos usar exclusivamente em modo teste (sem dinheiro real).
Criar conta no Stripe

Acesse o link e preencha: email → nome completo → país (Brasil) → senha

Confirme o email clicando no link enviado

No dashboard, você já está em modo teste

Aparece "Test mode" no topo com um toggle roxo. Isso é exatamente o que queremos!

Vá em "Developers" (menu lateral) → "API keys"

Anote duas chaves:

• Publishable key — começa com pk_test_...

• Secret key — começa com sk_test_... (clique em "Reveal test key" para ver)

!

Importante: NÃO ative o modo live enquanto estiver no curso. Tudo será em modo teste. Nenhum dinheiro real vai circular.

"Stripe criado, modo teste ativo. Nenhum dinheiro real vai circular enquanto a gente desenvolve."

Conta online

Resend — Emails Transacionais

O que é: Resend envia emails transacionais (convites de membros, confirmações). Interface moderna, API simples. Plano free: 3.000 emails/mês.
Criar conta no Resend

Acesse o link e clique em "Sign up with GitHub"

Mais rápido, aproveita a conta que já criamos.

Autorize o acesso

No dashboard, vá em "API Keys" (menu lateral)

Clique em "Create API Key":

• Name: pipeflow-crm

• Permission: Full access

• Domain: Sending (padrão)

Copie a API key gerada (começa com re_...)

!

Atenção: Essa chave só aparece UMA VEZ! Cole imediatamente num bloco de notas e salve.

i

O Resend dá um domínio gratuito para testes: onboarding@resend.dev. Perfeito pra começar.

"Resend criado, API key salva. Vamos usar isso lá no módulo de convites de membros."

Vercel — Deploy e Hospedagem

O que é: Vercel é onde o app vai ficar online. A cada push no GitHub, ela faz deploy automaticamente. Literalmente: deu push, tá no ar em 30 segundos.

Autorize o acesso

Escolha o plano Hobby (gratuito)

Perfeito pro curso. Inclui deploy ilimitado, HTTPS automático e domínio .vercel.app

✓

Você cai no dashboard. Não precisa criar projeto agora — faremos quando conectar ao repositório.
