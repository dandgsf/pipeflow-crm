Chaves, clients e conexão.
O projeto encontra o Supabase.

Até agora trabalhamos com dados fake. Agora é pra valer. Primeiro passo: conectar o projeto ao Supabase com as credenciais certas.

O que essa aula constrói

Quatro peças. Conexão real.

Project URL, chaves, arquivo protegido e dois clients. Tudo que o projeto precisa pra conversar com o Supabase.

🔑

Chaves do Supabase

Project URL, anon key e service_role key. Três credenciais com papéis diferentes.

🛡

Arquivo .env.local

Onde ficam as chaves. Nunca vai pro GitHub — protegido pelo .gitignore.

🌐

Browser Client

Conexão do navegador com o Supabase. Usa a anon key com proteção RLS.

⚙

Server Client

Conexão do servidor com o Supabase. Acesso com cookies e async.

Planejar → Construir → Testar → Iterar
Os prompts da aula

Do planejamento ao commit

Prompt 01 — Planejamento

Criar a branch e alinhar com o Claude o que vai ser feito.

Leia o CLAUDE.md e o PLAN.md. Estamos na aula 3.1 — Setup Supabase & Chaves. Crie a branch feat/supabase-core e me descreva o que vai implementar.
Prompt 02 — Construção

O prompt principal — cria as chaves, o .env e os dois clients.

Crie o .env.local com as chaves do Supabase e os clients conforme o PLAN.md: client.ts (browser, lazy singleton) e server.ts (servidor, cookies, async). Confirme que o .env.local está no .gitignore.
Prompt 03 — Revisão

Claude revisa o próprio trabalho antes do teste humano.

Revise o que você criou. Confirme que o .env.local tem as 3 chaves, que está protegido no .gitignore, que os dois clients existem e que o build passa sem erros.
Prompt 04 — Commit e finalização

Salvar, abrir PR, fazer merge e limpar.

Faça o commit, abra o PR pra main, faça o merge e delete a branch.
