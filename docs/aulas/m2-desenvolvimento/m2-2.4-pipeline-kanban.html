O drag-and-drop usa a biblioteca @dnd-kit — o Claude Code instala e configura tudo automaticamente. Você só pede "arrastar e soltar" no prompt.

Os prompts da aula

O pipeline nasce de 3 prompts

Prompt 01 — Planejamento

Criar branch e pedir pro Claude usar a skill de design.

Leia o CLAUDE.md e o PLAN.md. Estamos na aula 2.4 — Pipeline Kanban UI. Crie uma branch pra essa feature. Use a skill frontend-design e se inspire no Pipedrive como referência visual. Me descreva o que vai implementar.
Prompt 02 — Construção

O prompt completo com tudo que o pipeline precisa ter.

Implemente o Pipeline Kanban conforme o PLAN.md com dados fake: - Board com 6 colunas (as etapas do pipeline) e scroll horizontal - Cada coluna mostra o total de negócios e valor em R$ - Cards dos negócios com título, lead, valor, responsável e prazo - Arrastar e soltar cards entre colunas (drag-and-drop) - Formulário pra criar novo negócio - Visual bonito, inspirado no Pipedrive. Use a skill frontend-design.
Prompt 03 — Revisão e commit

Testar o drag-and-drop e salvar.

Revise o pipeline. Testa se o drag-and-drop funciona, se os totais das colunas atualizam, se dá pra criar novo negócio. Faz o commit, abre PR pra main, faz merge e deleta a branch.
!

O drag-and-drop é a parte mais complexa do app inteiro. Se o Claude gerar algo que não funciona de primeira, é normal. Mostra o erro, pede pra corrigir. Faz parte do ciclo de iteração.

Planejar → Construir → Testar → Iterar
