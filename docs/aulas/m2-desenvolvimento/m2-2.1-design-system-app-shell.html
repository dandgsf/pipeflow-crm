Conceito fundamental

Git: a máquina do tempo do projeto

Antes de começar a construir, entenda como o Git protege seu trabalho. Três conceitos. Três analogias. Zero complicação.

📸

Commits = Fotos do Projeto

Cada commit é uma foto do projeto naquele momento. Se algo quebrar, volta pra foto anterior. Simples assim.

commit 1 — "layout criado"
commit 2 — "sidebar pronta"
commit 3 — "algo quebrou" ← volta pro 2
🌿

Branches = Rascunhos Isolados

main = versão publicada. Branch = rascunho separado. Se o rascunho der errado, a versão publicada continua intacta.

main
feature
↑ merge de volta
✅

Pull Request = Pedido de Aprovação

Rascunho pronto? Abra um Pull Request — é um pedido pra juntar o rascunho de volta na versão estável.

📝
1. Cria a branch e desenvolve
🔍
2. Abre o PR pra revisão
✅
3. Aprova e faz o merge na main

Quem faz tudo isso? O Claude Code. Você pede em português. Ele executa. O pensamento é: isolar → salvar progresso → revisar → integrar.

Onde acompanhar?
No Cursor (IDE)

Controle de código-fonte

Na barra lateral, o ícone de controle de código-fonte (parece uma ramificação). Ali você vê arquivos alterados, commits e a branch atual.

No GitHub

Pull Requests e histórico

Quando o Claude Code faz push e abre o PR, tudo aparece no GitHub: commits, arquivos alterados, e você pode aprovar.

Na prática

Os prompts que você vai usar

Quatro prompts. Copie, cole no Claude Code e acompanhe o resultado. O ciclo completo: planejar, construir, revisar, integrar.

Prompt 1

Planejamento + Branch

Leia o CLAUDE.md e o PLAN.md. Estamos na aula 2.1 — Design System e App Shell. Crie uma branch pra essa feature e me descreva o que vai implementar.

O Claude lê o plano do projeto, cria uma branch separada e descreve o que vai fazer antes de começar.

Prompt 2

Execução

Implemente o esqueleto visual do PipeFlow conforme o PLAN.md: sidebar com menu de navegação, barra superior, seletor de workspace com dados fake, dark mode como padrão, e componentes base que vamos reutilizar nas próximas aulas. Sidebar que vira menu hamburguer no celular.

Um único prompt e o Claude Code constrói toda a estrutura visual. Sidebar, topbar, workspace switcher, dark mode, responsivo.

Prompt 3

Revisão

Revise o que você criou. Confere se a sidebar funciona em mobile, se a navegação troca entre as páginas e se o dark mode ta aplicado direitinho.

O Claude revisa o próprio trabalho: checa responsividade, navegação e visual antes de você testar.

Prompt 4

Commit + PR

Faça o commit dessa feature, abra um PR pra main, faça o merge e delete a branch.

Git completo pelo Claude Code: commit, push, Pull Request, merge e limpeza. Você só acompanha.

Route Groups

As pastas com parênteses organizam as rotas sem afetar a URL. Cada grupo define seu layout.

    (app)/ ← área logada com sidebar
    (auth)/ ← telas de login
    (marketing)/ ← landing page
🌙

Dark Mode

O app usa dark mode. A landing page usa light mode. Cada route group define seu visual.
