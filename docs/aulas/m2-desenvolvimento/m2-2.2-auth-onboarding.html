Login, cadastro e primeiro acesso.
Tudo visual.

Formulários com validação, loading nos botões e mensagens de erro. Mas o login é fake — na FASE 2 conecta de verdade.

O que essa aula constrói

Três telas. Uma experiência completa.

O usuário abre o app e vê login, cadastro e onboarding. Tudo bonito e funcional visualmente — mas sem conexão com banco. Ainda.

🔑

Página de Login

Email e senha, validação nos campos, botão com loading e mensagem de erro estilizada.

📝

Página de Cadastro

Nome, email, senha e confirmação. As mesmas validações do login, com feedback visual.

🏢

Onboarding

Após o cadastro, o usuário dá nome ao primeiro workspace e é redirecionado ao dashboard.

⚠ Navegação fake

O botão de login só redireciona pro dashboard — não valida email nem senha de verdade. O cadastro também não salva nada no banco. Isso é proposital: primeiro a gente constrói a experiência visual, depois conecta tudo ao Supabase Auth na FASE 2.

Planejar → Construir → Testar → Iterar
Os prompts da aula

O que você pede pro Claude Code

Quatro prompts. Do planejamento ao commit. Você direciona, o Claude executa.

Prompt 01 — Planejamento

Primeiro, o Claude lê os arquivos do projeto e planeja o que vai fazer.

Leia o CLAUDE.md e o PLAN.md. Estamos na aula 2.2 — Auth & Onboarding UI. Crie uma branch pra essa feature e me descreva o que vai implementar.
Prompt 02 — Construção

Agora sim: pedir pra construir as três telas com tudo que precisam.

Implemente as telas de login, cadastro e onboarding conforme o PLAN.md. Formulários com validação nos campos, loading nos botões e mensagens de erro. Por enquanto a navegação é fake — login redireciona pro dashboard sem validar de verdade. O onboarding pede o nome do primeiro workspace e redireciona pro dashboard.
Prompt 03 — Revisão

Antes de finalizar, o Claude revisa o próprio trabalho.

Revise o que você criou. Verifica se os formulários têm validação, se os botões têm loading, se as mensagens de erro aparecem, e se a navegação tá redirecionando certo. Roda o projeto e testa.
Prompt 04 — Commit e PR

Tudo validado? Hora de salvar e mandar pro repositório.

Faça o commit dessa feature, abra um PR pra main, faça o merge e delete a branch.
i

Lembrete: o ciclo é sempre Planejar → Construir → Testar → Iterar. O prompt 03 é a parte de Testar — e o prompt 04 só roda quando tudo tá funcionando.
